## החלטות עיצוב שהתקבלו

### סטטוסים
```
ממתין לליקוט → בליקוט → בדרך ללקוח → סופק
```
- מסעדן רואה: **"בדרך אליך"** (במקום "בדרך ללקוח")
- כל שאר השחקנים: **"בדרך ללקוח"**
- חשוב: עקביות מלאה בשמות בכל הממשקים

### ניווט
- ישראל → **Waze**
- גרמניה + הולנד → **Google Maps**

### עריכת הזמנה
- וואטסאפ: כפתור "ערוך הזמנה" — טקסט חופשי או הקלטה קולית
- אפשר גם להתקשר מחדש ולהקריא שינויים לבוט
- לאחר עריכה: מגיעה הודעת אישור עם ההזמנה המעודכנת

### פרטי נהג למסעדן
- שם הנהג + מספר טלפון מוצגים **רק** כשסטטוס עובר ל"בדרך אליך"

### חשבוניות
- מחירים מגיעים מ-API לתוכנת הנהלת החשבונות (שלב 6)
- צריך מילון מוצרים — מיפוי בין שם שהבוט מזהה לשם המדויק בנה"ח
- שני סוגי חשבוניות: נפרדת לכל הזמנה + חודשית מאוחדת (לפי בחירת המסעדן)
- תשלום דרך האפליקציה — בשלב מאוחר יותר

### מלקט שהוא גם נהג
- רואה שתי לשוניות (ליקוט + משלוח) תמיד
- לאחר "הזמנה לוקטה" → המערכת שואלת: "האם אתה הנהג?" → אם לא, בוחר מהרשימה
- לשונית משלוח מתאכלסת אוטומטית לאחר סיום ליקוט

### צ'קליסט ליקוט
- כפתור "הזמנה לוקטה" נעול עד שכל הפריטים מסומנים
- יש כפתור "פריט חסר" לדיווח חריגים (לוגיקה עדיין לא אופיינה)

## מה הלאה — סדר עדיפויות

### שלב הבא מיידי: אפיון בוט קולי
- ממשק ניהול מילון מוצרים (ממשק למנהל)
- תרחישי שיחה — שיחה רגילה / שיחה עם שינוי / שיחה בשפה זרה
- מה קורה כשהבוט לא מבין? (בקשת חזרה / העברה לנציג?)
- איך מזהים מסעדן חדש לעומת חוזר?

### אחרי הבוט: פיתוח MVP
**סדר פיתוח מוצע:**
1. בוט קולי (Twilio / Vonage + Whisper לתמלול + GPT להבנה)
2. שמירת הזמנה ב-DB + שליחת וואטסאפ (Twilio WhatsApp API)
3. ממשק מסעדן (React Native / PWA)
4. דשבורד מלקט/נהג
5. דשבורד מנהל
6. API נהלת חשבונות

### טכנולוגיות שדיברנו עליהן
- ניווט ישראל: **Waze Deep Link**
- ניווט חו"ל: **Google Maps URL**
- וואטסאפ: **Twilio WhatsApp API** (או 360dialog)
- מחירים: **API לתוכנת הנה"ח** (עדיין לא הוחלט איזו תוכנה)
