# PRD — מערכת הזמנות חכמה לליקוט ומשלוח

**גרסה:** 0.1 — טיוטת אפיון ראשונית  
**תאריך:** אפריל 2026  
**סטטוס:** בפיתוח — אפיון ועיצוב הושלמו, פיתוח טרם החל

---

## 1. סקירה כללית

מערכת B2B לניהול הזמנות סחורה ממסעדות וגופי מזון.  
המסעדן מתקשר למספר טלפון → בוט קולי חכם קולט את ההזמנה → המערכת מנהלת את כל שרשרת הליקוט, המשלוח, החשבונות והגביה.

### קהל יעד
- **מסעדנים** — לקוחות החברה. רוב האוכלוסייה אינה טכנולוגית, ממשק חייב להיות פשוט ביותר.
- **מלקטים ונהגים** — עובדי החברה, עובדים עם אפליקציה במהלך יום העבודה.
- **מנהל החברה** — ממשק ניהולי מלא לצפייה, בקרה וגביה.

### שפות נתמכות
עברית, אנגלית, גרמניה (בשלב ראשון). הבוט לומד ניבים ומבטאים.

## 3. מילון סטטוסים אחיד

> חשוב: שמות הסטטוסים חייבים להיות אחידים בכל המערכת.  
> ההבדל היחיד: המסעדן רואה "בדרך אליך", כולם האחרים רואים "בדרך ללקוח".

| סטטוס במערכת | מה המסעדן רואה | מה הצוות/מנהל רואים |
|---|---|---|
| `WAITING_PICK` | ממתין לליקוט | ממתין לליקוט |
| `PICKING` | בליקוט | בליקוט |
| `IN_DELIVERY` | בדרך אליך | בדרך ללקוח |
| `DELIVERED` | סופק | סופק |

## 4. שלב 1 — בוט קולי

### תיאור
מספר טלפון ייעודי שעונה אוטומטית 24/7.  
המסעדן מקריא את רשימת המוצרים שלו + מועד איסוף מבוקש.  
הבוט מתמלל, מבין, ושומר הזמנה מובנית במערכת.

### יכולות הבוט
- זיהוי שפה אוטומטי (עברית / אנגלית / גרמנית)
- הבנת ניבים ומבטאים שונים
- זיהוי שמות מוצרים גם אם נאמרים בצורות שונות (מיפוי למילון מוצרים)
- זיהוי כמויות
- זיהוי מועד איסוף מבוקש
- למידה לאורך זמן — זיהוי דפוסי הזמנה של לקוח קבוע
- אפשרות להקריא שינוי בשיחה — הוספה / הסרה / תיקון לפריט

### מילון מוצרים
כל מוצר שהבוט מזהה ממופה לשם המדויק בתוכנת הנהלת החשבונות.  
זהו תנאי הכרחי לסנכרון מחירים אוטומטי בשלב 6.

### פלט
```json
{
  "restaurant_id": "rest_123",
  "requested_pickup": "2026-04-18T08:00:00",
  "items": [
    { "product_id": "p_001", "name": "עגבניות שרי", "qty": 4 },
    { "product_id": "p_002", "name": "חסה איסברג", "qty": 6 }
  ],
  "raw_transcript": "...",
  "language_detected": "he"
}
```

## 5. שלב 2 — אזור מסעדן

### 5.1 הודעת וואטסאפ אוטומטית

**נשלחת מיד עם קליטת ההזמנה:**

```
שלום [שם מסעדה]!
קיבלנו את הזמנתך ל-[תאריך].

פירוט:
• [פריט] × [כמות]
• ...

איסוף מבוקש: [שעה]

[כפתור: אשר הזמנה]
[כפתור: ערוך הזמנה]
```

**אפשרויות עריכה:**
1. כתיבה חופשית בטקסט ("הסר גבינה, הוסף 2 לימונים")
2. הקלטה קולית
3. התקשרות חוזרת והקראת שינויים

לאחר עריכה → נשלחת הודעת אישור עם ההזמנה המעודכנת.

### 5.2 ממשק אזור אישי למסעדן

**עקרון עיצוב:** פשוט ביותר. כפתורים גדולים. טקסט ברור. מינימום בלבול.

#### לשונית: ההזמנות שלי
- פילטר חודשי (ברירת מחדל = החודש הנוכחי)
- כל הזמנה בשורה: תאריך + מספר פריטים + סטטוס צבעוני
- לחיצה על שורה = פירוט מלא

#### לשונית: מעקב הזמנה פעילה
ציר זמן עם 4 נקודות:
1. התקבלה ואושרה ✓
2. בליקוט (+ שם המלקט)
3. בדרך אליך (+ שם הנהג + טלפון + צפי הגעה)
4. סופק ✓

> פרטי הנהג (שם + טלפון) מוצגים רק כשהסטטוס עובר ל"בדרך אליך"

#### לשונית: חשבוניות

**תצוגת "לפי הזמנה":**
- פילטר חודשי
- כרטיסי סיכום: סה"כ החודש + ממתין לתשלום
- כל חשבונית: תאריך + מספר + סכום + סטטוס (שולם / ממתין)
- לחיצה = פירוט: פריט / כמות / מחיר ליחידה / סה"כ
- כפתור הורדת PDF
- הערה: "מחירים מסונכרנים מתוכנת הנהלת החשבונות"

**תצוגת "חודשי מאוחד":**
- חשבונית אחת לכל חודש המאחדת את כל ההזמנות
- פס ויזואלי של נפח ביחס לחודשים אחרים
- לחיצה = פירוט מלא

**תשלום:** בשלב מאוחר יותר — כרגע מוצג סטטוס בלבד, תשלום מחוץ למערכת.

## 6. שלב 3 — ניהול צוות

### הרשמת עובד
- שם, טלפון, תפקיד (מלקט / נהג / מלקט+נהג)
- שיוך מסעדות מתוך רשימת הלקוחות
- המערכת זוכרת את השיוך ומפנה הזמנות אוטומטית

### כללים
- עובד יכול לקבל עד 2 תפקידים
- כשמלקט גם נהג — הוא רואה את שתי הלשוניות תמיד
- כשמסיים ליקוט → המערכת שואלת: "האם אתה גם הנהג?" → אם לא, בוחר נהג מהרשימה

## 7. שלב 4 — דשבורד מלקט/נהג

### לשונית: ליקוט
**הזמנה פעילה (אחת בכל פעם):**
- שם מסעדה + שעת איסוף
- צ'קליסט פריטים — סימון כל פריט שנאסף
- פס התקדמות (X מתוך Y פריטים)
- כפתור "פריט חסר" לדיווח חריגים
- כפתור "הזמנה לוקטה" — מופעל רק כשכל הפריטים מסומנים

**הזמנות ממתינות:**
- תצוגה מקדימה של פריטים
- כפתורי "התחל ליקוט" ו"פירוט"

### לשונית: משלוח
- מופיעה אוטומטית לאחר סיום ליקוט
- שם מסעדה + כתובת
- מספר תעודת משלוח + כמות פריטים
- כפתור "סופק" — מעדכן סטטוס לכל השחקנים
- כפתור "ניווט" — פותח Waze לכתובת המסעדה (ישראל = Waze, גרמניה = Google Maps)

### לשונית: הושלם
- היסטוריית הסיומות של היום
- פריטים עם סימון strikethrough

## 8. שלב 5 — דשבורד מנהל החברה

### כרטיסי סיכום (ראש הדף — תמיד גלויים)
| ממתין לליקוט | בליקוט | בדרך ללקוח | סופק היום |
|---|---|---|---|
| מספר | מספר | מספר | מספר |

### לשונית: הזמנות
- פילטר לפי סטטוס: הכל / ממתין לליקוט / בליקוט / בדרך ללקוח / סופק
- כל הזמנה: מסעדה + תאריך + פריטים + עובד משויך + שעת איסוף + סכום + סטטוס
- לחיצה = פירוט מלא

### לשונית: מסעדות
- חיפוש חופשי
- כל מסעדה ניתנת לפתיחה עם:
  - סטטיסטיקות: הזמנות שהושלמו / פעילות כרגע / חוב פתוח
  - הזמנות אחרונות עם סטטוס
  - כפתורים: כל ההזמנות / חשבוניות / עריכה

### לשונית: גביה
- פילטר: הכל / חוב פתוח / שולם
- כרטיס לכל מסעדה: סכום חוב + מספר חשבוניות פתוחות + תאריך הזמנה אחרונה
- כפתור "שלח תזכורת" — שולח וואטסאפ אוטומטי למסעדן
- כפתור "סמן כשולם"
- סיכום תחתון: סה"כ חובות פתוחים

### לשונית: צוות
- כל עובד: שם + תפקיד + סטטוס פעיל/לא פעיל
- הזמנות היום + כמה סופקו + מסעדות משויכות
- ניהול שיוך מסעדות לעובד

## 9. שלב 6 — הנהלת חשבונות אוטומטית

### חיבור API
- סנכרון מחירים ממוצרים לפי מילון המוצרים
- הנפקת חשבונית אוטומטית עם אישור ההזמנה על ידי המסעדן
- עדכון סטטוס תשלום אוטומטי

### חשבונית אוטומטית
תיווצר עם:
- פרטי מסעדה מלאים
- פירוט פריטים + כמויות + מחירים מסונכרנים
- מספר חשבונית סדרתי
- תאריך + מועד פירעון

## 12. עקרונות עיצוב

1. **פשטות מעל הכל** — רוב המסעדנים אינם אנשים טכנולוגיים
2. **כפתורים גדולים** — ממשק מובייל first
3. **סטטוס צבעוני ברור** — צהוב = ממתין, כחול = פעיל, ירוק = הושלם, אדום = בעיה
4. **הודעות וואטסאפ** — ערוץ תקשורת ראשי עם המסעדן
5. **אחידות מינוח** — אותו שם סטטוס בכל מקום (למעט ההבחנה בדרך אליך / בדרך ללקוח)
